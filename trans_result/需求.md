# 需求分析文档

## 一、总体目标

通过学习 drpy-node 项目的框架源码和现有的 187 个 JS 源，掌握 drpy DS 源的编写方法论，
输出系统性学习总结，并基于此优化现有的 4 个 skill，大幅提升 AI 写源能力。

## 二、需求拆解

### 2.1 学习阶段（训练）

#### 2.1.1 框架核心源码学习
- **目标文件**：`libs/drpyS.js`、`libs/drpysParser.js`、`libs_drpy/template.js`
- **关键理解点**：
  - 沙箱（sandbox）隔离机制：`vm.createContext` + `vm.Script`
  - 模块初始化流程：`init()` → `handleTemplateInheritance()` → `initParse()`
  - 模板继承机制：rule.模板 → `template.getMubans()` → `Object.assign` 合并
  - 调度分发机制：`invokeMethod()` → 根据 method 类型和 rule 字段类型（string/function/js:/*）分流
  - 字符串规则解析：`commonCategoryListParse`、`commonClassParse`、`commonDetailListParse` 等
  - 播放链路：`lazy` 的 `parse:0/1`、`jx:0/1` 返回值语义
- **学习产出**：理解 drpy 框架的设计哲学和运行机制

#### 2.1.2 源文件风格学习
- **目标目录**：`spider/js/` 下 187 个 JS 源
- **重点分析维度**：
  - **模板继承型**（如 `樱花动漫[优].js`）：极少量代码，`模板:'mxpro'` 覆盖个别字段
  - **字符串规则型**（如 `DJ音乐[听].js`）：CSS 选择器字符串描述列表/详情
  - **函数驱动型**（如 `独播库[优].js`）：全 async function，API 签名逻辑
  - **混合型**（如 `360影视[官].js`）：部分字段用字符串，复杂逻辑用函数
  - **网盘型**（如 `网盘[模板].js`）：多网盘转存/解析逻辑
  - **模板基础文件**（`_base.js`、`_base1[画].js`、`_base2[画].js`）
  - **工具库文件**（`_lib.*.js`、`_fq.js`、`_360.js`、`_qq.js`）
- **学习产出**：掌握各种风格的最佳实践和适用场景

### 2.2 技能优化目标

#### 2.2.1 现有 4 个 skill 分析
当前 skill 列表：
| Skill | 职责 | 质量评估 |
|-------|------|---------|
| `drpy-node-source-create` | 从零新建源 | 良好，已有模板/API/SPA 三类路线 |
| `drpy-node-source-workflow` | 修源/测试/上传总控 | 良好，分流机制清晰 |
| `drpy-node-play-debug` | 播放链排障 | 良好，决策树完整 |
| `drpy-node-repo-upload` | 仓库上传 | 良好，红线机制明确 |

#### 2.2.2 优化方向
1. **加深框架理解**：现有 skill 偏重"怎么做"，缺少"为什么"。需补充框架内部机制。
2. **补齐模板系统知识**：当前 skill 提到了模板继承但未深入 template.js 的 12+ 种模板细节。
3. **补齐 drpy API 知识**：利用 MCP 的 `get_drpy_libs_info`、`get_drpy_api_list` 增强知识面。
4. **补齐高级用法**：`hostJs`、`预处理`、`模板修改`、`filter` 压缩编码等。
5. **MCP 能力深挖**：理解 MCP 源码中各工具的边界，更好配合使用。
